NPM

You can use NPM as a 'shopping' for code packages.

You do it through the terminal. To install a jquery and a normalize.css package you need to insert the following commands:

  • Navigate to the folder you want to work on and command npm init to start npm. This will create a package.json file in the directory. In this file you can see all the dependencies. Its our 'grocery' list where we keep track of all the extensions what we install through npm in our project.
  • npm install jquery or replace jquery by any other package of choice. The installation will create the 'node-modules' folder and inside it the 'jquery' one. This will only install the package, but to make the installations organized and easy to be tracked, do it as follows:
  • install jquery --save-- save will save the information that you installed jquery in your repo adding this information to the package.json file, under 'dependencies' (not the package-lock.json one). There you will also see the version of the package(s) which was installed. This file will save all the information on all the packages which were installed in a repo. After each installation, always add --save to keep this file up to date.
  • Also, in case the packages have been deleted, you can also install everything again if you only have the package.json file. For that, inside the repo type npm install. That will install all directories with the packages files again, which were lost.
  • After installing all the needed packages, rememeber to push them to the server. Just make sure to add the package.json file to the repo, add it to the staging area, commit it and push it to my github account.
  • if you want to check the version of the package which you have installed, just type npm jquery -v. Or replace 'jquery' by the desired package
  • In order to know which package to install look in npm website or google the action a package should perform followed by 'npm'
  • Together with jquery, when starting a project, install also the normalize package (equivalent to reset in css): npm install normalize.css --save

BEM

  • BEM stands for Block Element Modifier
  • Avoid cascading, so elements can be used directly without confusion
  • CSS target at elements directly
  • Tipp: translateY(y) - position an emlement vertically relative to itself. If for example, translateY(100) is used, tha will push the content down 100% of the total height of the content. So if the content is 200px tall, then it will be lowered 200px. If translateY(50) is used for a 200px height content, then the content will go 50% down, or 100px.
  • Font size remthe browser looks at what is the font size defined for the main <html> element for that user's browser. Most browsers have a default of 16px for html, but users can change browser's settings individually. When, for instance 2rem is defined as font size of a class, the font size of this class is twice as big as the font-size of the overal html element, which turns out as 32px for most browsers. But if some users configured their browsers to have a different font size, with rem, we respect the user's font size of their browser so that the proportions between the elements is the same although not all browsers of all users have a 16px font size.

    Git Branches

    • git branch shows the branches in the repo
    • create a new branch with each new feature of the project
    • clean project before creating new branch: check if there are no uncommited changes
    • Now add all files for being commited!git add folder/file.extension for individual file or git add -A If I only want to remove one file from the list, git reset folder/file.extension
    • check status git status
    • commit git commit -m 'insert comment of what you did in the new commited files'
    • Ig all worked fine, you sill see the 'nothingto commit' comment, as below:
    • create a new branch git branch name-of-branch. Choose a name which make sense according to the changes done.
    • After these command lines go to the new branch: git checkout name-of-branch
    • Make changes needed and then commit them: git commit -m 'add comment'
    • Switch back to the master branch git checkout master to see that here the changes of the other branch didnt occurr
    • After finished with the changes in the branch, to merge them, go to master branch and do: git merge name-of-branch
    • Push master branch to github git push origin master
    • git commit -am Git add + git commit -m will just commit those files you've added (new and previously tracked), but git commit -am will commit all changes on tracked files, but it doesn't add new files.
    • Its also possible to merge a branch in github directly. For that, create the branch in github: git push origin name-of-branch
    • Go to github, see your branch and assign it to yourself or a colleague. Once you did this, you can approve the merge in github. There will be two branches visible on your profile. After you accept the merge, the branch will be empty and you should delete the empty branch, as a best practice.
    • Go to the master branch git checkout master and pull the changes done in the master in github (the merging) into your local workspace, otherwise you wont see the new changes.
    • Check your braches and delete possible empty branches: git branch shows you how many branches you have
    • Delete empty branches: git branch -d name-of-branch
    • git checkout -b new-branch creates a new branch and goes directly to it
